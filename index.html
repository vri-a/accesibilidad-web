¿Cómo
crear un sitio web más accesible?

La accesibilidad web refiere a hacer que un sitio web sea accesible para todos, especialmente para los usuarios con discapacidades o capacidades diferentes. Para lograr esto, se fomenta el uso de diseño inclusivo, tecnología integrada y software.

Algunos simples cambios en nuestro código HTML

HTML Semántico

El HTML semántico es un concepto para estructurar y escribir código HTML de modo que el significado del contenido de una página web sea claro a través de sus etiquetas.

Usar HTML semántico significa que al escribir el código, todo el contenido buscará llevar la etiqueta más adecuada según su tipo.

Los lectores de pantalla para usuarios ciegos o con baja visión organizan la lectura del contenido según la estructura del código. Entonces un código con buenas prácticas y semántica ayuda a que el sitio sea adecuado para audiencias con diferentes necesidades.

Botón con la etiqueta div

La etiqueta HTML <button> está diseñada para ser accesible desde el teclado.

Imágenes

Las imágenes y los videos pueden crear barreras para los usuarios cuando no son accesibles. Todas las imágenes deben tener el atributo alt donde se debe agregar una descripción adecuada, según el tipo de imagen.

Tipos de imagenes

Decorativas:

Las imágenes decorativas se agregan con fines estéticos. El atributo alt debe quedar vacío, alt="" para que los lectores de pantalla puedan ignorarlas.

Funcionales:

Las imágenes funcionales son aquellas que se incluyen en enlaces, botones y otros elementos interactivos. El atributo alt debe llevar información sobre el propósito del enlace.

alt="Ir a nuestro perfil de Instagram"

Informativas:

Las imágenes informativas representan un concepto o información. El atributo alt debe contener el mismo texto que la imagen.

Enlaces

Los enlaces se utilizan para crear un enlace a otras páginas, dentro de la misma página, a otros recursos, etc.

Para los enlaces:

Proporciona una descripción que indique el propósito del enlace.

Usa las pseudoclases hover y focus para resaltarlos mejor.

Darle al enlace un color diferente que al resto del texto no siempre es suficiente, ya que no todas las personas pueden percibir los colores de la misma manera. Subrayar los enlaces permite una mejor visualización de los mismos.

Clic aquí

Evita descripciones como "Clic aquí", "Ver más", etc.

Formularios

Los formularios permiten la interacción y el manejo de la información.

Para los formularios:

Deben ser fáciles de usar, claros e intuitivos.

Operables por el teclado.

Cada input debe tener su etiqueta label asociada

Para navegar en el teclado se utilizan principalmente las siguientes teclas:

Tab y Shift + Tab para moverse por todos los elementos que pueden ser activados.

Flechas direccionales para moverse dentro de un elemento.

Enter o espacio para activar o seleccionar un elemento.

No eliminés el contorno (outline) ya que los usuarios del teclado perderán su orientación en la página.

Algunos cambios en nuestro código JavaScript

ARIA

ARIA (Accessible Rich Internet Applications) es un conjunto de roles y atributos que definen formas de hacer contenido web y aplicaciones web (especialmente aquellas desarrolladas con JavaScript) más accesibles para personas con discapacidades..." MDN Web Docs

Un ejemplo de ARIA con JavaScript

Los elementos dinámicos cambian de estado, por ejemplo, un menú desplegable se puede contraer o expandir. ARIA te permite definir las propiedades y estados de los elementos.

Cuando tengás menús desplegables en tu web, debés cambiar dinámicamente el estado usando JavaScript para que las tecnologías de apoyo puedan transmitir el cambio al usuario.

Algunos estados ARIA

  • aria-busy
  • aria-checked
  • aria-expanded
  • aria-hidden
  • aria-disabled

aria-expanded = false

Algunos sitios y herramientas útiles

Pautas de Accesibilidad al Contenido en la Web (WCAG)

Son recomendaciones para crear contenido accesible. Ir al sitio de la WCAG

Servicio de validación de marcado del W3C.

Verifica que el contenido HTML no tenga errores. Si la estructura HTML tiene errores, las diferentes tecnologías de asistencia no pueden representarlo correctamente. Servicio de validación de marcado

Herramientas de evaluación automática

Lighthouse

Podés usar Lighthouse desde DevTools para auditar la accesibilidad de tu página y generar un informe.

WAVE

WAVE facilita la evaluación humana y educa sobre accesibilidad.

Accessibility Insights

Accessibility Insights ayuda a los desarrolladores a encontrar y solucionar problemas de accesibilidad en sitios y aplicaciones web.

Funkify

Funkify actúa como un simulador de discapacidad y modifica tu sitio para que puedas verlo como lo ven otras personas.
